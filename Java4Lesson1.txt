1. Как вы собираетесь передавать файлы?
   - Сериализация

2. Где вы собираетесь их хранить?
   На сервере будет создана папка с названием логина пользователя.
   Внутри папки данные будут храниться в базе данных.

3. Нужна ли база данных и зачем?
   Нужна
   Будет 4 БД.
   1. Общая БД с данными о пользователях
      Логин
      Пароль
      Размер хранилища
      Размер фрейма
      Количество доступных потоков для многопоточной загрузки
   2. Общая БД для хранения прав доступа пользователей к хранилищу
      Логин хранилища
      Логин которому разрешен доступ
      Права доступа:
          	OWNER - собственник хранилища
		    GUEST - минимальные полномочия
		    READ - только чтение
		    READADD - чтение и добавление новых файлов(изменние запрещено)
		    READWRITE - полный доступ на изменение хранилища.
   3. Личная для каждого пользователя БД с данными о файлах
      ID файла
      Собственник хранилища (возможно избыточно но пусть)
      Кто загрузил/изменил
      Имя файла
      Полный путь к файлу(структура каталогов при загрузке на сервер с клиента)
      Время создания файла(оригинальное)
      Время загрузки на сервер
      Размер файла
      Количество загруженых фреймов
      Статус файла:
      		FINISHED (файл полностью загружен и доступен для пользователей)
       		UPLOADING (файл находится в процессе загрузки)
        	ARCHIVED
        	DELETED
      Код для доступа по ссылке
      Время действия кода
   4. Личная для каждого пользователя БД с самими файлами
      Файлы будут храниться в виде фреймов фиксированного размера.
      		ID файла
      		Номер фрейма
      		Данные
   *. Возможно БД для логгирования событий.

4. Что вы хотите передавать помимо файлов?
	Взаимодействие клиент/сервер будет осуществляться через сообщения:
		Регистрация пользователей
		Аутентификация пользователей
		Запросы на синхронизацию
		Данные синхронизации
		Запросы на модификацию файлов
		Фреймы данных
	Все будет последовательно.

5. Библиотеки для работы?
	Пока что вполне достаточно java.io
	Для БД MySQL
